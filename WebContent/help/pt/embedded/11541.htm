Configurar processos de Signoff

A seguir é esclarecido como você deve configurar um Processo de Signoff. No ARIS, você pode modelar os objetos relevantes para Signoff, alterando o procedimento conforme esclarecido a seguir.

Pré-requisito

Você tem o papel Gestor de Signoff.

Procedimento

  1. Crie grupos de usuários para cada nível de hierarquia e atribua a cada um o papel Gestor de Signoff.
  2. 2. Defina um Gestor de Signoff para cada nível de hierarquia, ou seja, os usuários que executam o processo de avaliação de cada nível de hierarquia. Atribua estes ao respectivo grupo de usuários, de modo que cada nível de hierarquia seja testado pelo respectivo usuário com o papel Gestor de Signoff.
  3. Defina a hierarquia a ser usada como base para o processo de Signoff. Na criação dos elementos hierárquicos, ative a opção Relevante para Signoff, para que eles sejam envolvidos no processo de Signoff e atribua no campo Gestor de Signoff o respectivo grupo de usuários.
  4. Atribua aos elementos hierárquicos os elementos básicos desejados: Risco ou Definição de teste. Com a atribuição dos elementos básicos, os casos de teste destes estão ligados com o elemento hierárquico. Estes casos de teste são verificados pelo Gestor de Signoff.
  5. Crie uma tarefa de Signoff.
  6. Gere o processo de Signoff.

O processo de Signoff pode ser executado.
