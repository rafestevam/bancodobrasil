O que é processo de signoff?

O processo de Signoff é um processo de avaliação. O processo de avaliação é um processo de várias etapas, permitindo que, ao longo de diferentes níveis de hierarquia, os elementos hierárquicos individuais sejam avaliados. Geralmente os resultados dos casos de teste que foram realizados dentro do período de controle servem de base para as avaliações. Os elementos básicos Risco, Controle e Definição de testes, por sua vez, servem de base para os casos de teste.

A avaliação dentro de um processo de Signoff é realizada a partir do nível mais baixo até o nível mais alto da hierarquia. Isto significa que a avaliação de um elemento hierárquico superior só pode ser realizada quando todos os subordinados já foram avaliados.

Assim que a avaliação de um elemento hierárquico subordinado está concluída, o grupo diretamente superior na hierarquia de Signoff é informado sobre a alteração de status. Ao mesmo tempo, os casos de teste e as deficiências do elemento hierárquico inferior são bloqueados.

Quando todos os elementos hierárquicos subordinados estiverem concluídos, o elemento hierárquico superior é liberado para o processo de ativação. Ou seja, o processo de Signoff de elementos hierárquicos superiores só pode iniciar, quando os seus elementos hierárquicos subordinados tenham sido totalmente testados.

Apenas o elemento hierárquico de Signoff superior deve ser atribuído a um Gestor de Signoff. Se não houver proprietários de Signoff atribuídos a elementos hierárquicos de Signoff subordinados, estes são liberados automaticamente pelo sistema para edição posterior.

Dentro do nível de hierarquia podem existir Elementos hierárquicos relevantes para Signoff e Elementos hierárquicos não relevantes para Signoff. Com isso, é possível que, através da avaliação de um elemento hierárquico relevante para Signoff, o processo de Signoff do elemento hierárquico superior já seja liberado, mesmo que no nível de hierarquia atual existam outros elementos hierárquicos.
